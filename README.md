# ⚓ Тренажёр ISSA Inshore Skipper + SRC (Telegram-бот)

Телеграм-бот для подготовки к теоретическому тесту **ISSA Inshore Skipper** и
**SRC (VHF Radio Operator)** на русском языке. **500 вопросов**, конспект,
шпаргалки, словарь яхтсмена и тренажёр задач на поправки компаса. Вопросы
**не повторяются**, пока не пройдёшь весь круг.

> ⚠️ **Важно.** Бот и база вопросов — **оригинальные**, созданы для тренировки.
> Они **не содержат экзаменационных ключей** и **не являются официальным
> материалом ISSA** или какой-либо школы. Это вспомогательный учебный материал
> для самостоятельной подготовки. Вопросы составлены по открытым первоисточникам
> (COLREG/МППСС-72, IALA MBS, GMDSS/SRC, стандартная навигация).

## Что умеет бот

| Режим | Описание |
|-------|----------|
| 🎯 **Случайный вопрос** | Подряд новые вопросы из всего банка, без повторов |
| 📚 **Тренировка по темам** | Выбор темы (COLREG, VHF, IALA, Огни, Навигация…) и тренировка только по ней |
| 📝 **Экзамен** | 100 вопросов, проходной балл 75%, в конце — результат и разбор ошибок (длинный разбор автоматически режется под лимит Telegram) |
| 🔁 **Работа над ошибками** | Повтор только тех вопросов, где вы ошибались (исправились → вопрос уходит из списка) |
| 🧭 **Решение задач (T-V-M-D-C)** | 50 задач на поправки компаса; ввод ответа числом, пошаговое решение. Есть задачи **со звёздочкой** (с подвохом — переход через 360°). Ведётся **серия (streak)**, рекорд и статистика по задачам |
| 📖 **Конспект** | Теория по 10 темам (устройство яхты, курсы, навигация, IALA, МППСС, огни, метео, VHF/SRC, безопасность), с **иллюстрациями** (схема яхты, курс, T-M-C, девиация, дневные знаки, шкала Бофорта) |
| 📌 **Шпаргалки** | 9 кратких выжимок: T-V-M-D-C, МППСС, огни, звуки, буи IALA, Бофорт, **узлы (с пошаговыми фото)**, швартовки, VHF/Mayday |
| 📚 **Словарь яхтсмена** | 180+ морских терминов простыми словами в 10 категориях, с **поиском** (по русскому/английскому/польскому) и польско-русским мини-словарём |
| 📊 **Статистика** | Сколько отвечено, % правильных, пройдено из банка |

У вопросов с расширенным материалом под ответом есть кнопка **«📖 Подробнее»**
(показывает поле `study` — углублённое пояснение).

Вопросы показываются как **нативные Telegram-quiz**: нажимаешь вариант — бот сразу
подсвечивает правильный/неправильный ответ и даёт **пояснение**.

## Установка и запуск

> Требуется **Python 3.10+** (проверено на 3.12). `aiogram` уже умеет работать на Windows.

```powershell
# 1. Перейти в папку бота
cd "c:\Users\user\Desktop\ISSA\bot"

# 2. (рекомендуется) создать виртуальное окружение
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Получить токен бота:
#    - открыть в Telegram @BotFather
#    - /newbot -> задать имя и username
#    - скопировать выданный токен

# 5. Создать файл .env (скопировать из .env.example) и вписать токен:
#    BOT_TOKEN=123456789:AA...ваш_токен

# 6. Запустить
python bot.py
```

После запуска откройте своего бота в Telegram и отправьте `/start`.

> ⚠️ Чтобы бот мог фиксировать ответы на quiz, опросы отправляются **не анонимно**
> (`is_anonymous=False`) — это требование Telegram для получения ответов. В личной
> переписке имя видит только сам бот.

## Как добавить свои вопросы

### Вариант A — через CSV (с автопроверкой дубликатов)

Подготовьте CSV с заголовком:

```csv
topic,question,A,B,C,D,answer,explanation
VHF / SRC,Какой канал для DSC?,70,16,13,72,A,DSC использует канал 70.
```

`answer` — буква `A/B/C/D` (или индекс). Затем:

```powershell
python import_csv.py путь\к\новым_вопросам.csv
```

Вопросы с уже существующим текстом **пропускаются** — повторов не будет.

### Вариант B — вручную в `questions.json`

Добавьте объект в массив:

```json
{
  "id": "vhf-019",
  "topic": "VHF / SRC",
  "q": "Текст вопроса?",
  "options": ["Вариант 1", "Вариант 2", "Вариант 3", "Вариант 4"],
  "answer": 0,
  "expl": "Пояснение к правильному ответу."
}
```

`answer` — индекс правильного варианта (0 = первый).

### Вариант C — объединить несколько баз с дедупликацией (`merge_dedup.py`)

Если у вас несколько CSV-баз (из разных школ) и их нужно слить в одну без
повторов — положите все `.csv` в папку `sources/` и запустите:

```powershell
python merge_dedup.py            # объединить + записать questions.json + отчёт
python merge_dedup.py --dry-run  # только анализ и отчёт, без записи
python merge_dedup.py --collapse-semantic  # ещё и свернуть смысловые дубли
```

Инструмент делает **три уровня проверки дубликатов**:

1. **Точные дубли** — одинаковый текст вопроса (после нормализации регистра,
   пунктуации, пробелов). Удаляются автоматически; `study_material`/пояснение
   подтягиваются из лучшего варианта.
2. **Близкие по тексту** — почти одинаковая формулировка (схожесть ≥ 0.88,
   алгоритм `SequenceMatcher`). Помечаются в отчёте.
3. **Смысловые дубли** — *один и тот же факт, разная формулировка*. Группируются
   по «подписи факта» (номер правила МППСС, комбинация огней, паттерн гудков,
   кардинальный знак, определяемый термин, числа в навигационной задаче).
   Например, «Что требует Rule 5?» и «Что капитан обязан вести всегда — зрением
   и слухом?» получают одну подпись `colreg:rule5-lookout`, хотя текст разный.

Также отчёт отдельно показывает **конфликты** — когда один и тот же вопрос в
разных базах имеет *разный правильный ответ* (требует ручной проверки).

Полный отчёт пишется в `dedup_report.txt`. По умолчанию смысловые дубли только
**помечаются** (решение за вами); с флагом `--collapse-semantic` из каждой
группы остаётся один лучший вопрос (с пояснением/study_material).

## Хранение прогресса

Прогресс каждого пользователя — в `progress/<user_id>.json`:

- `seen` — какие вопросы уже показаны (логика «без повторов»);
- `wrong` — вопросы для «работы над ошибками»;
- `stats` — общая статистика.

Сбросить можно кнопкой «♻️ Сбросить прогресс» в меню (или см. `/privacy`).

> При перезапуске бота **сохранённый прогресс не теряется**. Сбрасываются только
> сессии, активные в момент перезапуска: незавершённый экзамен, текущая задача
> и режим поиска по словарю (всё это пока хранится в памяти).

Бот работает **только в личном чате**: прогресс и quiz привязаны к одному
пользователю. В группах бот вежливо отправляет в личку.

## Развёртывание на сервере

Бот работает на long-polling — внешний адрес/домен не нужен. Подробная
пошаговая инструкция для Linux-VPS (Ubuntu) с автозапуском через `systemd`:
см. **[`deploy/DEPLOY.md`](deploy/DEPLOY.md)**.

Кратко:

```bash
cd /opt && git clone <repo> issa-bot && cd issa-bot
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
echo 'BOT_TOKEN=...' > .env && chmod 600 .env
cp deploy/issa-bot.service /etc/systemd/system/
systemctl daemon-reload && systemctl enable --now issa-bot
```

Обновление: `git pull` → `systemctl restart issa-bot` (или `bash deploy/update.sh`).
Папку `progress/` стоит включить в резервное копирование.

## Тесты и качество

- `python run_audit.py` — пред-релизный чек-лист (валидность банка, картинки,
  лимиты Telegram, дубли, `.env` в `.gitignore` и т.д.).
- `python test_nav.py` — тесты решателя T-V-M-D-C (вкл. границу 0°/360°).
- `python test_load.py` — нагрузка 10 одновременных пользователей.

CI (GitHub Actions, `.github/workflows/ci.yml`) гоняет всё это на каждый push,
плюс **secret-scan** на случай утечки токена.

  морских правил (COLREG/МППСС-72, IALA MBS, процедуры VHF/GMDSS) — см.
  `../_extracted/sources.txt`.

Бот и база вопросов оригинальны, не содержат экзаменационных ключей и не являются
официальным материалом ISSA или какой-либо школы.
